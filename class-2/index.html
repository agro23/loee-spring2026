Before App 2: your Anthropic API key

App 2 uses the Anthropic API to summarize articles. You need an API key, which is different from your Claude Pro subscription.

  1. Go to console.anthropic.com in your browser.
  2. Sign in or create an account.
  3. Go to Settings → API Keys → Create Key.
  4. Copy the key (starts with sk-ant-…) and keep it somewhere handy.
Keep your API key safe. Don't paste it into a public website or commit it to a GitHub repo. For class, pasting it into a local file on your own computer is fine.

App 1 prompt
Build an Etsy-style product marketplace using HTML, CSS, and JavaScript in a single index.html file. Include:

- A homepage with a grid of 8 fictional product cards. Each card shows a product image placeholder (use emoji or colored div), product name, seller name, price, and a short description.
- A category filter bar at the top with at least 4 categories (e.g., Art, Jewelry, Home, Vintage). Clicking a category filters the grid.
- A detail view that appears when a product card is clicked, showing larger info and a "Buy Now" button (the button just shows an alert for now).
- Warm, handcrafted styling with a creme background, serif headings, and rounded cards. Make it feel like a local artisan market.

Keep everything in one file. No external dependencies.

When Claude Code finishes, run a local server to view your app:

Run local server
npx serve .

Open the URL shown in the terminal (usually http://localhost:3000).

Try an iteration

  • "Change the categories to books, records, plants, and coffee."
  • "Add a search bar at the top that filters products by name."
  • "Make the background a soft lavender and change the font to something modern."

App 2 prompt
Build an "AI News Watchdog" as a single-file HTML web app. The app should:

- On first load (or when the user clicks a "Start Watching" button), fetch the current top stories from the Hacker News API (https://hacker-news.firebaseio.com/v0/topstories.json and the individual item endpoint).
- Filter the stories to find ones with "AI", "LLM", "Claude", "GPT", "ML", or "model" in the title (case-insensitive).
- For each matched story found, display a card showing the title, the HN link, and a placeholder for a summary.
- For the summary, call the Anthropic API (claude-haiku-4-5-20251001 model) with the story's title and ask Claude to write a 2-sentence summary of what the story is likely about based on the title alone.
- After the initial load, continue polling Hacker News every 30 seconds for new matching stories. Track which story IDs you've already processed so you don't duplicate work.
- Show a status indicator at the top: "Listening for new AI stories..." with a pulsing animation.
- The API key should be loaded from a hardcoded variable at the top of the file (we'll paste it in).
- Use clean, modern styling with a dark mode aesthetic.

Keep everything in one index.html file. Use the fetch API, no build tools.
  1. When the file is generated, open index.html and find the line where the API key goes. Paste your key there (inside the quotes).
  2. Run the app:
Run local server
npx serve .

Open the URL. The app should fetch the top AI story from Hacker News and summarize it. Leave the tab open — it'll keep watching for new stories.

Honest framing: this runs in your browser tab. When you close the tab, the watchdog stops. In a production version, you'd move this exact code to a server that runs 24/7. Same logic, different home.

App 3 prompt
Build a Pokemon card viewer as a single-file HTML web app. The app should:

- Have a search input where the user types a Pokemon name (e.g., "pikachu", "charizard", "mewtwo").
- When the user submits, fetch from the PokeAPI (https://pokeapi.co/api/v2/pokemon/{name}) and display a card showing the Pokemon's official artwork sprite, name, types (with colored type badges), height, weight, and base stats (HP, Attack, Defense, Special Attack, Special Defense, Speed) as horizontal bars.
- Show a loading state while fetching and a friendly error message if the Pokemon isn't found.
- Style it like a trading card: rounded corners, drop shadow, bright colors matching the Pokemon's type, and a subtle gradient background.
- Include a few example names as clickable suggestions below the search input to help users get started.

Keep everything in one index.html file. No API key needed.
  1. Run the app:
Run local server
npx serve .

Type a Pokemon name and see what happens. Try: pikachu, charizard, mewtwo, eevee, snorlax.

Troubleshooting

ProblemFix
"npx is not recognized"Install Node.js first: nodejs.org. Restart your terminal after installing.
Page is blank when I open index.html directlyUse npx serve . in the project folder instead of double-clicking the file. That starts a local web server.
App 2 doesn't fetch anythingCheck: did you paste your Anthropic API key in the right spot? Did you keep the quotes around it? Is the key still valid (no extra spaces)?
App 2 shows a CORS errorRun the app with npx serve . instead of opening the file directly. CORS needs a real HTTP server.
App 3 shows "Pokemon not found"Check your spelling. All lowercase. Try pikachu (not Pikachu).
Claude Code is stuck thinking foreverCtrl+C to cancel. Try a simpler prompt. If it's still stuck, restart Claude Code.
API rate limit errorWait 60 seconds and try again. If it persists, raise your hand.
Anything elseRaise your hand. Andy's your person.
