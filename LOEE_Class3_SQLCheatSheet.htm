SQL CHEAT SHEET

The 11 Commands You’ll Actually Use

LOEE Class 3 · Keep this on your desk

Claude Code writes SQL for you. But knowing the basics lets you read what it wrote, spot mistakes, and write your own quick queries in the Supabase SQL Editor.

 

The Commands

SELECT * FROM tasks;

Get all rows from the tasks table.

SELECT title, created_at FROM tasks;

Get only the title and created_at columns.

SELECT * FROM tasks WHERE completed = true;

Filter rows. Note: true/false are unquoted.

SELECT * FROM tasks WHERE title = 'Buy milk';

Filter by string. Single quotes for strings.

SELECT * FROM tasks ORDER BY created_at DESC;

Sort by created_at, newest first. ASC for oldest first.

SELECT * FROM tasks ORDER BY title ASC LIMIT 10;

First 10 alphabetically. LIMIT caps the rows returned.

SELECT COUNT(*) FROM tasks;

Count rows. Returns a single number.

SELECT COUNT(*) FROM tasks WHERE completed = true;

Count only matching rows.

INSERT INTO tasks (title) VALUES ('Walk the dog');

Add a new row. Other columns get default values.

UPDATE tasks SET completed = true WHERE id = 5;

Change rows. ALWAYS use a WHERE clause.

DELETE FROM tasks WHERE id = 5;

Remove rows. ALWAYS use a WHERE clause or you delete everything.

 

 

Gotchas (Read This)

 

1. Single quotes for text. Never double quotes.

In SQL, double quotes are for column or table names that have spaces in them — which we avoid. Single quotes are for the actual text values you're storing or comparing.

RIGHT:  WHERE title = 'Buy milk'

WRONG:  WHERE title = "Buy milk"     -- SQL thinks "Buy milk" is a column name

 

2. Don't forget WHERE on UPDATE and DELETE.

If you forget the WHERE clause, you change or delete EVERY row. The SQL editor will warn you, but if you click through the warning, your data is gone.

DANGEROUS: DELETE FROM tasks;             -- removes ALL rows

SAFE:      DELETE FROM tasks WHERE id = 5; -- removes ONE row

 

3. Column names are case-insensitive (mostly).

title and TITLE refer to the same column. By convention, use lowercase with underscores: created_at, user_id, not CreatedAt or userID.

 

4. NULL is not the same as empty string or zero.

NULL means "no value" or "unknown." Comparing to NULL with = doesn't work the way you'd expect.

WRONG: WHERE user_id = NULL          -- returns nothing

RIGHT: WHERE user_id IS NULL         -- works

RIGHT: WHERE user_id IS NOT NULL     -- works

 

5. Strings are case-sensitive in comparisons.

'Buy milk' is not equal to 'buy milk'. If you want to ignore case, use ILIKE:

WHERE title = 'buy milk'      -- exact match only

WHERE title ILIKE 'buy milk'  -- case-insensitive match

WHERE title ILIKE '%milk%'    -- contains 'milk', any case

 

6. Semicolons end statements.

In the Supabase SQL Editor, you can run multiple statements at once if you put a semicolon after each. Forgetting a semicolon between statements causes errors.

 

7. ASC is the default for ORDER BY.

ORDER BY title is the same as ORDER BY title ASC. You only need to specify if you want DESC.

Reading What Claude Code Generated

When Claude Code writes SQL for you, look for these patterns:

 

CREATE TABLE — making a new table

CREATE TABLE table_name (

  column_name TYPE constraints,

  another_column TYPE constraints

);

Each line inside the parentheses is one column. The TYPE is what kind of data it holds (TEXT, BOOLEAN, BIGSERIAL, TIMESTAMPTZ, UUID). Constraints add rules (NOT NULL, DEFAULT something, PRIMARY KEY).

 

CREATE POLICY — row-level security rule

CREATE POLICY "policy_name" ON table_name FOR action

  USING (condition);

"action" is SELECT, INSERT, UPDATE, or DELETE. "condition" is what must be true for the user to be allowed to do the action. auth.uid() is the current user's ID.

 

ALTER TABLE — changing an existing table

Used to add columns, drop columns, enable RLS, etc. Common pattern:

ALTER TABLE tasks ENABLE ROW LEVEL SECURITY;

ALTER TABLE tasks ADD COLUMN priority INTEGER DEFAULT 0;

 

If You Get Stuck

Two things to try when SQL fails:

1.    Read the error message. Postgres errors are usually specific: "column 'title' does not exist" tells you the column name is wrong. "syntax error near..." tells you exactly where the parser choked.

2.    Paste the SQL and the error into Claude Code. Ask it to fix it. This is faster than debugging yourself.
