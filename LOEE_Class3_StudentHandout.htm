Step 1: Create the Tasks Table

In the Supabase dashboard, click SQL Editor in the left sidebar. Paste and run:

CREATE TABLE tasks (

  id BIGSERIAL PRIMARY KEY,

  title TEXT NOT NULL,

  completed BOOLEAN DEFAULT false,

  created_at TIMESTAMPTZ DEFAULT now(),

  user_id UUID REFERENCES auth.users(id)

);

 

What this means:

•      id — unique number for each task, generated automatically.

•      title — the task description text. NOT NULL means it's required.

•      completed — true or false. Defaults to false (not done yet).

•      created_at — when this task was added. Filled in automatically.

•      user_id — which user owns this task. Will tie tasks to accounts.

 

Click Run. Switch to Table Editor. Your empty tasks table is there.

 

Step 2: Lock It Down with Row-Level Security

Without security, anyone can read anyone's tasks. We'll fix that.

 

1.    In the Supabase dashboard, find the AI assistant (chat icon).

2.    Ask it: "Generate RLS policies for the tasks table that let users read, insert, update, and delete only their own tasks based on the user_id column."

3.    Review the SQL it generates. Apply it.

 

Or paste this directly:

ALTER TABLE tasks ENABLE ROW LEVEL SECURITY;

 

CREATE POLICY "select own tasks" ON tasks FOR SELECT

  USING (auth.uid() = user_id);

 

CREATE POLICY "insert own tasks" ON tasks FOR INSERT

  WITH CHECK (auth.uid() = user_id);

 

CREATE POLICY "update own tasks" ON tasks FOR UPDATE

  USING (auth.uid() = user_id);

 

CREATE POLICY "delete own tasks" ON tasks FOR DELETE

  USING (auth.uid() = user_id);

 

Now the database itself enforces "only see your own tasks." Even if your app code has a bug, the database refuses unauthorized requests.

Step 3: Wire Up the Starter App

Open the starter app folder you downloaded. In your terminal, navigate into it. Then start Claude Code:

cd path/to/starter-app

claude

 

Paste this prompt:

Wire-Up Prompt:

"Take this index.html starter and connect it to Supabase. Add a sign-in/sign-up form using Supabase Auth (email + password). Replace the in-memory tasks array with calls to a Supabase tasks table. The table has columns: id, title, completed, created_at, user_id. When the user is signed in, show their tasks. When they sign out, show the sign-in form. Use the Supabase JS client loaded from the CDN. I'll paste my project URL and anon key into placeholders at the top of the script."

 

Find Your Supabase URL and Anon Key:

1.    In Supabase: Project Settings (gear icon) > API.

2.    Copy the "Project URL."

3.    Copy the "anon public" key. NOT the service_role key — that one bypasses security.

 

Paste both into the placeholders Claude Code added to your file. Save.

 

Run the app:

npx serve .

 

Sign up. Add a task. Refresh the page. The task is still there — because it's in your real database now.

 

Step 4: Add Real-Time Updates

Make the app update across devices instantly when data changes.

 

1.    In Supabase: Database > Replication. Enable replication for the tasks table.

2.    Back in your project folder, run claude again.

3.    Paste this prompt:

 

Real-Time Prompt:

"Add a Supabase real-time subscription to the tasks table so that when a task is inserted, updated, or deleted in the database, the UI updates automatically without refreshing. Use the channel API from supabase-js."

 

Save. Run npx serve. Open the app in two browser windows side by side. Sign in as the same user in both. Add a task in one. Watch it appear in the other instantly.

Troubleshooting

 

Problem

Fix

Email confirmation not arriving for signup

Supabase: Authentication > Settings > toggle off "Confirm email" for class. Re-enable it later for production.

"Permission denied" when querying tasks

RLS is working as designed. You're not signed in, or signed in as a different user. Sign in first.

App shows blank after wire-up

Check browser console (F12). Most likely: URL or anon key wasn't pasted correctly. Make sure you used the anon key, not service_role.

Tasks don't appear in real-time

Did you enable replication for the tasks table in Database > Replication?

"npx serve" not found

Install Node.js: https://nodejs.org. Restart terminal after installing.

CORS errors

Run via npx serve, not by opening the file directly.

Anything else

Raise your hand. Andy is your person.

Adapt the Template

The skeleton you built today works for any "users have private data" app. Try one of these:

•      Notes app: replace tasks with notes (id, title, content, user_id).

•      Recipe collection: title, ingredients, instructions, user_id.

•      Habit tracker: name, last_done_at, streak, user_id.

•      Reading list: book_title, author, status, rating, user_id.

 

Each one is the same architecture with different column names. That's the takeaway from today: you've learned a pattern, not just a single app.
