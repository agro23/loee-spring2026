Put Your Project Online

Follow these steps to deploy your project and get a shareable link.

What does "deploy" mean?

Right now, your project only exists on your laptop. Deploying means putting it on the internet so anyone with the link can see it — your coworkers, your family, your friends. You'll get a URL like yourproject.vercel.app that works on any device.

We're using a free service called Vercel. It hosts your site and gives you a link. The whole process takes about 2 minutes.

Step 1: Create a free Vercel account

Go to vercel.com/signup in your browser. You can sign up with:

  • Your email address (simplest option)
  • Your GitHub account (if you have one)
  • Your Google account

Pick whichever is easiest. You just need to be able to log in.

Already have a Vercel account? Skip to Step 2.

Step 2: Install the Vercel tool

In your terminal, run this command:

Install Vercel CLI
npm install -g vercel

This installs Vercel's deployment tool on your computer. You only need to do this once.

Then log in:

Log in
vercel login

This opens your browser to verify your account. Follow the prompts, then come back to the terminal.

Step 3: Deploy your project

Navigate to your project folder in the terminal. If you're not sure where it is, ask Andy. Then run:

Deploy
vercel

Vercel will ask you a few questions. Here's what to answer:

Vercel asks you…You answer…
Set up and deploy?Y (yes)
Which scope?Select your personal account
Link to existing project?N (no — create a new one)
What's your project's name?Type your first name (e.g., jane-loee)
In which directory is your code?Press Enter (use the default: ./)
Want to modify build settings?N (no)

Vercel will upload your project. When it's done, you'll see something like:

✅ Production: https://jane-loee.vercel.app [12s]

That URL is your live site. Open it in your browser. Text it to someone. It's on the internet right now.

Step 4: Update your site (optional)

Made changes to your project with Claude Code? Deploy the update:

Deploy an update
vercel --prod

This pushes your latest version to the same URL. Your link stays the same — the content updates.

Shortcut: skip the questions

If you want to skip all the prompts and just deploy fast, run:

Deploy with defaults
vercel --yes

This accepts all the defaults. Great for quick updates.

Something not working?

ProblemWhat to do
"command not found: vercel"Close and reopen your terminal, then try again. If still broken, run npm install -g vercel again.
"Error: No framework detected"That's OK for a static site. Vercel should still deploy. If it asks for a framework, choose "Other".
Deploy succeeded but page is blankCheck that your project has an index.html file in the root folder. If your files are in a subfolder, you may need to cd into it first.
"You have exceeded the free tier"Unlikely for this class, but if it happens, ask Andy.
